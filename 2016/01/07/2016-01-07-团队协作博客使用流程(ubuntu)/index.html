团队协作博客使用流程(ubuntu)

使用说明
  • blog
  • github
  • hexo
  • joe

本文只是讲解如何使用博客进行记录我们学习的点滴,不会讲博客的搭建流程。用博文进行进行文章的记录，只是我们的一种学习方式，工具用着顺手方便即可，没有必要纠结博客的搭建过程，有兴趣可以自行研究，但请勿本末倒置。

基本说明

博客书写规范说明

  1. 每篇博客明明规则为:日期+文章标题名,例如”2015-01-07-团队协作博客使用流程(ubuntu)”

  2. 写完博客请先进行本地调试,本地调试完成后再进行部署

  3. 博客代码原稿请push到coding(提交之前请先pull)

  4. 博客代码生成好的静态文件请部署到github

(规范正在整理中,将会在使用过程中不断完善……)

所需环境说明

  • Git（自行安装）
  • node.js
  • github账户（见群共享）
  • hexo
  • coding账号

环境配置

安装node.js

  1. 下载安装包 http://nodejs.org/dist/latest-v5.x/node-v5.3.0.tar.gz

  2. 解压到合适文件夹，例如 /usr/local/

  3. 设置全局链接

源码安装和apt-get安装方式详见http://my.oschina.net/blogshi/blog/260953

安装hexo

  1. 输入以下命令进行hexo安装
    npm install -g hexo
  2. 本地测试环境
  • 新建一个文件夹，例如test
  • 进入文件夹，输入hexo init
  • 安装 node_modules
    输入命令sudo npm install，该命令会自动在文件夹中生成node_modules文件夹
  • 启动服务器
    输入hexo server启动服务器，然后浏览器访问http://localhost:4000/,可以看到自己网站，说明环境已经配置好

在github上配置ssh-key

因为hexo只支持ssh方式进行部署，不支持https方式进行部署，所以我们需要在github账号中配置我们的ssh-key

  1. 切换到root账户，输入ssh-keygen -t rsa -C "your email"
    （目前尚不清楚为什么使用sudo hexo deploy命令部署时，系统会读取/root/账户根目录下的公钥文件，倘若直接在当前目录下进行公钥生成，那么部署将会失败）

  2. 然后一直敲回车键

  3. 输入cat ~/.ssh/id_rsa.pub查看公钥，然后复制公钥到“右上角账户->setting-> SSH Keys”即可

写我们自己的博客

首次进行博客书写

  1. 输入如下命令将coding上的代码clone到本地
    git clone https://git.coding.net/undefines/cilab_undefine_blog.git
    coding上是我们博客的源码，github上的博客只是我们编译好之后的静态文件
  2. 安装node_modules模块
    sudo npm install
    coding上同步的文件夹不包含node_modules模块，所以首次下载需要自行安装
  3. 新建博客
    输入hexo new "your new article name",然后系统会自动在source/_post/文件夹下创建一篇名字为your new article name.md的文章
    ,然后在该md格式文档中书写内容即可。
  4. 编译文章
    输入hexo generate,nodejs会自动编译博客
  5. 本地预览
    输入hexo server命令启动本地服务器，然后访问http://localhost:4000/即可以看到自己的文章的效果
  6. 推送到github
    输入git deploy即可以完成到github的部署
  7. 推送到coding
    推送到coding,便于小组其他人进行协作

非首次进行博客书写

注意: 以上流程讲解的是首次进行博客写作的时候的流程,如果不是首次进行写作(即本地有coding仓库代码),请按照如下流程

  1. 新建博客
  2. 本地调试
  3. pull coding代码(因为其他人对博客可能有改动,导致本地代码库有可能不是最新版本)
  4. 部署到github
  5. 同步推送到coding
